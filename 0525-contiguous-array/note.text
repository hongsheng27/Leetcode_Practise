LeetCode 525 - Contiguous Array

題目：給一個只有 0 和 1 的陣列，找出 0 和 1 數量相等的最長連續子陣列長度。

解題思路：Prefix Sum + HashMap（560 的變形）
第一個要解決的問題：怎麼把「0 和 1 數量相等」變成數學條件？
直接比數量很難處理。
關鍵轉換：把 0 換成 -1。
這樣每個 1 貢獻 +1，每個 0 貢獻 -1。
「0 和 1 數量相等」= 「子陣列和 = 0」。

問題變成 560 的變形：找和為 0 的最長子陣列。
第二個要解決的問題：怎麼找和為 0 的子陣列？
子陣列和 = prefix[j] - prefix[i]。
和為 0 代表 prefix[j] == prefix[i]。
也就是說：如果兩個位置的 prefix sum 相同，中間那段的和就是 0。
所以用 HashMap 記錄每個 prefix sum 第一次出現的位置。

第三個要解決的問題：為什麼存 index 而不是次數？
560 要算「幾個」子陣列 → map 存次數。
這題要算「最長」子陣列 → map 存 index。
而且只存第一次出現的位置，因為第一次離當前最遠 = 子陣列最長。
遇到重複的 prefix sum 不更新 map，直接算長度。

第四個要解決的問題：補 0 技巧。
count = {0: -1} 的意思是「prefix sum = 0 出現在 nums 開始之前，記為 index -1」。
沒有它會漏掉從頭開始的子陣列。
例子：[0, 1]，整個陣列和 = 0，答案是 2。
走到 r=1 時 curSum = 0，查 map 找到 index -1，長度 = 1 - (-1) = 2。
如果 map 裡沒有 {0: -1}，curSum = 0 不在 map 裡，就漏掉了。
跟 303 的 prefix = [0, ...] 和 560 的 {0: 1} 是同一件事：
告訴程式「在最開頭，prefix sum 就是 0」。

複雜度：O(n) 時間，O(n) 空間。
