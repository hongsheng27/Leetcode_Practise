88. Merge Sorted Array

題目概述： 題目：將兩個已排序的陣列 nums1 和 nums2 合併。nums1 的長度為 m + n，前 m 個是有效元素，
後 n 個預留為 0；nums2 長度為 n。請將結果直接合併到 nums1 中，不回傳任何值。

解題邏輯： 三個指標從尾端開始，比較 nums1[p1] 和 nums2[p2]，大的放到 nums1[p3]。主迴圈跑到其中一邊用完，再用第二個迴圈把 nums2 剩餘搬進去。nums1 剩餘不用管，因為本來就在正確位置。
利用「已排序」特性：因為兩者原本就是有序的，不需重新發明排序演算法。逆向填入：從 nums1 的最末端（最大的位置）開始往前回填，
可以避免覆蓋掉尚未比較的有效數字。

踩過的坑：
1.一開始想把邊界處理塞進同一個迴圈，導致 p1 < 0 時 Python 負數索引不報錯但拿到錯誤值。拆成兩個迴圈最乾淨。
2.沒意識到 bottleneck 是 nums2——迴圈結束後問自己「誰剩下會造成結果不完整」，nums1 剩的本來就在原地，nums2 剩的沒搬就丟失。
3.Edge case nums1=[0], m=0, nums2=[1], n=1：p1 初始就是 -1。拆兩個迴圈時，p1 >= 0 是進入條件，第一個迴圈直接跳過，第二個迴圈安全搬完 nums2。單一迴圈寫法容易在這裡讓 -1 當索引存取到錯誤值。

複雜度： O(m+n) time / O(1) space
