LeetCode 88 - Merge Sorted Array
==============================
題目：將兩個已排序的陣列 nums1 和 nums2 合併。nums1 的長度為 $m + n$，前 $m$ 個是有效元素，
後 $n$ 個預留為 0；nums2 長度為 $n$。請將結果直接合併到 nums1 中，不回傳任何值。

核心概念：In-place（原地修改）：不能建立新陣列，必須直接修改傳入的 nums1 記憶體空間。
利用「已排序」特性：因為兩者原本就是有序的，不需重新發明排序演算法。逆向填入：從 nums1 的最末端（最大的位置）開始往前回填，
可以避免覆蓋掉尚未比較的有效數字。

解題思路：三指針法：p1 指向 nums1 的有效末端 ($m-1$)。p2 指向 nums2 的末末端 ($n-1$)。
p 指向 nums1 整個空間的最末端 ($m+n-1$)。執行邏輯：比較 nums1[p1] 和 nums2[p2]，誰大就把它填入 nums1[p]，然後將該指針與 p 同時前移。
收尾：如果 nums2 還有剩（p2 >= 0），代表這些剩下的數都比 nums1 裡面的數字小，直接把它們補到 nums1 前面即可。

踩過的坑：Python 變數指向問題：若寫成 nums1 = nums1[0:m] + nums2，Python 會在記憶體建立一個「新的 List」並讓變數名指向它，
但原本 LeetCode 測試系統持有的那個舊記憶體位址內容完全沒變。解決法：應使用 nums1[:] = ... 或是直接透過索引 nums1[p] = ... 修改。
單次遍歷排序的誤區：想說合併後只用一趟交換（類似 Bubble Sort 一次）就能排好。但排序需要多次比對才能確保中間的數字正確歸位。

核心理解：這題不是要你重新排序，而是要你「利用已排序的資訊」進行合併。

複雜度：
時間複雜度：$O(m + n)$（只需要走完兩個陣列的長度）。
空間複雜度：$O(1)$（直接使用預留空間，沒有額外空間開銷）。
