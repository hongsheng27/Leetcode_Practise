LeetCode 75 - Sort Colors

題目：給一個只有 0, 1, 2 的陣列，原地排好成 [所有 0 | 所有 1 | 所有 2]。

解題思路：Three-Way Partition（27 題 Two Pointer 的延伸）
第一個要解決的問題：怎麼從兩段分類變成三段？
27 題：一個條件，分兩段，兩個指針（慢指針標記寫入位置，快指針掃描）。
這題：兩個條件，分三段，需要三個指針。
l = 下一個 0 該放的位置（左邊界）
r = 下一個 2 該放的位置（右邊界）
i = 當前掃描位置
不是新概念，是同一個 pattern 加一個維度。
第二個要解決的問題：swap 完 i 要不要前進？
nums[i] == 0 → swap 到 l，換來的東西來自 l，而 l <= i，已經看過 → i 前進。
nums[i] == 2 → swap 到 r，換來的東西來自 r，還沒看過，可能是 0 或 2 → i 不動。
判斷原則：swap 完要不要移動指針，取決於「換來的東西看過沒」。
第三個要解決的問題：迴圈什麼時候停？
while i <= r，不是 i < r。
i == r 時那個元素還沒歸類，跳過就錯了。

操作流程：
nums[i] == 0 → swap(l, i)，l 和 i 都前進
nums[i] == 1 → 不動，i 前進
nums[i] == 2 → swap(r, i)，r 後退，i 不動

複雜度：O(n) 時間，O(1) 空間。

踩過的坑：
while i < r 寫成嚴格小於 → i == r 時元素沒處理。
nums[i] == 2 時 i 也前進 → 從右邊換來的沒看過，會漏掉。

觸發關鍵字：
看到「原地分成兩段」→ two pointer（27 題）。
看到「原地分成三段」→ three-way partition（這題）。
跟 Quick Sort 的關係：
這題 pivot 固定是 1，Quick Sort 的 pivot 是動態選的，但三路操作完全一樣。
Quick Sort 碰到大量重複元素退化成 O(n²)，加上三路 partition 就能解決。
