LeetCode 236 - Lowest Common Ancestor of a Binary Tree 筆記
Code 
========================================================

題目：給一棵二元樹和兩個節點 p、q，找到它們的最低公共祖先（LCA）。

核心觀察：
  在當前節點做決定時，不知道 p 和 q 在哪邊，必須先遞迴左右子樹收集資訊再判斷。
  → 自然形成 post-order（先左右，再處理 root）。

  對比 235（BST 版本）：BST 的大小關係直接告訴你該往哪走，不需要子樹結果，
  所以是 pre-order，時間 O(h)。

解題思路：
  定義回傳值合約：
    回傳 None  → 這棵子樹裡 p 和 q 都不在
    回傳非 None → 找到了某個目標，或已經找到 LCA

  Base case：root 為空 → None；root 就是 p 或 q → 直接回傳 root。
  遞迴：往左右子樹各找一次。
  合併：
    左右都非 None → p 和 q 分居兩側 → root 就是 LCA
    只有一邊非 None → 答案在那邊，往上傳
    都是 None → 往上傳 None

踩過的坑：
  1. 合併邏輯太複雜：試圖追蹤找到的是 p 還是 q，導致分支爆炸。
     其實不需要知道找到的是誰，只需要知道「有沒有找到東西」，
     合併就簡化成 if left and right: return root / return left or right。
  2. 比較對象搞混：p、q 是 TreeNode，不是值，應該比 root == p 而不是 root.val == p。
     回傳也應該是節點本身（return root），不是 return root.val。
  3. 回傳值語意不一致：有時回傳 root.val、有時回傳 p、有時回傳 None，
     每個代表的意思不同，合併時自然打結。
     → 寫遞迴前先定義清楚「回傳值合約」，確保每個分支回傳的語意一致。

思考框架（適用所有樹的遞迴題）：
  問自己：「在這個節點做決定時，需不需要子樹的結果？」
    需要 → 先遞迴再判斷 → post-order
    不需要 → 當下就能決定方向 → pre-order
  Order 是思考邏輯的結果，不是起點。

複雜度：
時間 | O(n)，最差情況遍歷所有節點
空間 | O(n)，遞迴堆疊深度，最差情況為鏈狀樹
