LeetCode 271 - Encode and Decode Strings
題目：設計一組 encode / decode 函式，把字串陣列壓成一個字串，再還原回來。
不能有歧義，任何輸入都要能正確還原。

踩過的坑：

單用 delimiter 無法達成目的。
字串內容本身可能包含任何字元，選什麼 delimiter 都可能撞到。
例如用 ","：["a,b", "c"] → "a,b,c" → decode 出 ["a", "b", "c"] ❌
空字串和空陣列的邊界問題。
不同輸入 encode 後結果相同，無法區分：
["", ""] → "" / [""] → "" / [] → "" 三者撞在一起。
長度本身可能超過一位數。
"200abc..." 不知道長度是 2、20 還是 200，需要一個方式定位長度的結尾。

解題思路：長度前綴 Protocol（類似 HTTP Content-Length）
第一個要解決的問題：怎麼知道每段字串的範圍？
不靠 delimiter，改成在每個字串前面加上它的長度。
這樣就算內容包含任何字元，只要照長度取固定數量的字元就好。
第二個要解決的問題：怎麼處理空字串和空陣列？
空字串的長度是 0，所以會被記為 "0#"。
["", ""] → "0#0#" / [""] → "0#" / [] → ""，三者不再衝突。
第三個要解決的問題：怎麼定位長度數字的結尾？
用 "#" 隔開長度和內容，decode 時用 find('#') 找到分界點。
確定長度後，往後取固定數量的字元，不受內容影響。
第四個要解決的問題：記錄答案要高效。
encode 用 join 一次串接，避免反覆字串相加（Python 字串相加是 O(n)）。
decode 用雙指標 i, j 切割，i 指向長度開頭，j 指向 #，一路往前推進。
編碼格式：
每個字串 → "長度#內容"
encode: ["hello", "world"] → "5#hello5#world"
decode: 讀長度 → 跳過 # → 根據長度取字串 → i 移到下一段起點 → 重複
複雜度：O(n) 時間，n 為所有字串的總字元數。O(n) 空間，存放結果。
