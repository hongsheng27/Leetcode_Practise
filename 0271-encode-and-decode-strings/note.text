LeetCode 271 - Encode and Decode Strings
題目：設計一組 encode / decode 函式，把字串陣列壓成一個字串，再還原回來。
不能有歧義，任何輸入都要能正確還原。

踩過的坑：
1.單用 delimiter 不夠：字串內容可能包含任何字元，選什麼 delimiter 都可能撞到內容。要用 長度 + delimiter + 內容，
length 是「可信的邊界資訊」，不依賴 delimiter 本身判斷邊界。
2.空字串邊界：空字串長度是 0，記為 0#。["", ""] → "0#0#"、[""] → "0#"、[] → ""，三者不衝突。
decode 時 length=0，s[j+1 : j+1+0] 就是 s[j+1 : j+1]，Python slice 起點等於終點回傳 ""，不需要特判，自然處理。
3.decode 順序要跟 encode 一致：encode 是「先寫 length，再寫 #，再寫字串」，decode 就從左到右「先讀 length，再跳 #，再讀字串」。
不要反向找數字——曾經卡在「找到 # 再往回找數字」，但字串內容本身可能有數字，反查邏輯會出錯。encode/decode 要對稱，方向一致。
4.高效寫法：encode 用 join 一次串接，避免反覆字串相加（O(n²)）。decode 用雙指標 i, j：i 指向長度開頭，
j 找到 #，抓完字串後 i 直接跳到下一段起點。
5.decode 外層 while 要寫：while r < len(s)
不要寫：
while r <= len(s)
因為最後一組會在 r 還小於 len(s) 時處理完。
處理完後 r 會剛好等於 len(s)，下一輪自然跳出。

複雜度： O(n) time / O(n) space，n = 所有字串總長度
decode: time compacity => len(s), O(1) / f'{}', 0(n) 因為他要一個數字一個數字處理 / "".join() 複雜度也是o(n),一個數字一個數字處理
