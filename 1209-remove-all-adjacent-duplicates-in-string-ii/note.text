1209. Remove All Adjacent Duplicates in String II
題目解釋：
給定字串 s 和整數 k，重複執行：找到剛好 k 個連續相同字元並消除，消除後左右拼接可能產生新的連續重複，繼續消除直到不能為止。
注意是剛好消 k 個，不是 <= k 全消。

解題思路：
單純存 char 不夠，需要知道連續幾次 → stack 存 [char, count]
當前字元跟 stack top 相同 → count + 1，不同 → 新增一組
每一步統一檢查：count == k 就 pop
連鎖消除不用手動處理，pop 後下一個字元自然跟新的 top 比較
最後把 stack 裡的 char * count 拼接成結果

模擬 "abxxxby", k=3：
'a' → [(a,1)]
'b' → [(a,1), (b,1)]
'x' → [(a,1), (b,1), (x,1)]
'x' → [(a,1), (b,1), (x,2)]
'x' → count==3, pop! → [(a,1), (b,1)]
'b' → top 是 b, count+1 → [(a,1), (b,2)]
'y' → [(a,1), (b,2), (y,1)]
結果："abby"（bb 只有 2 個，不到 k=3，不消除）
邊界情境速查表（以 k=2 為例）：
"abbb"  →  消一對 bb → "ab"（3 個只能消 k 個，剩 1 個）
"aaa"   →  消一對 aa → "a"（不是全消成 ""）
"abba"  →  消 bb → "aa" → 消 aa → ""（連鎖消除自動發生）

踩過的坑：
1.想用雙指針 + while 連續消除 → 越想越複雜，因為消除後產生新的相鄰重複需要不斷回頭檢查。
2.Stack 天然解決：歷史狀態都在 stack 裡，pop 後自動跟新 top 比較, 但是使用 [char, count] 結構。
3.沒搞清楚「消除 k 個」的意思 → 是剛好消 k 個，不是 >= k 全消。判斷用 count == k，而非 count <= k。
4.return 用 join 比較好

複雜度（n = 字串長度）：
時間 | O(n)，每個字元最多進出 stack 各一次
空間 | O(n)，最差情況沒有任何消除，stack 存所有字元
