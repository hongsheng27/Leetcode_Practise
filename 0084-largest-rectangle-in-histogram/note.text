# LeetCode 84 - Largest Rectangle in Histogram

題目：給一排柱狀圖的高度，找出能框出的最大長方形面積。

關鍵觀察：最大長方形的高度，一定等於某一根柱子的高度。
所以對每根柱子問：「用我的高度當長方形的高，最多能往左往右延伸多遠？」
延伸規則：旁邊的柱子 >= 我的高度就能繼續，碰到比我矮的就停。

換句話說：
左邊界 = 往左找，第一根比我矮的柱子在哪
右邊界 = 往右找，第一根比我矮的柱子在哪
寬度 = 右邊界 - 左邊界 - 1

=== 1. 暴力解 O(n^2) ===

對每根柱子，往左掃、往右掃，找第一根比我矮的。
一根柱子最壞掃 2n 次 = O(n)，n 根柱子 = O(n^2)。

=== 2.Monotonic Stack O(n) ===
Stack 在這裡的用途：
Stack 幫你記住「還沒找到右邊界的柱子」。
暴力解裡每根柱子都自己往右掃去找第一根比自己矮的，但很多柱子可以一起等。
你從左往右走，碰到一根矮柱子的時候，stack 裡所有比它高的柱子同時找到答案。

例子：heights = [2, 5, 6, 2]
走到 index 1, 2 時高度遞增，沒人找到右邊界，先放進 stack 等：
stack: [2, 5, 6] ← 三根都還沒碰到比自己矮的
走到 index 3，高度 2，比 6 矮也比 5 矮：
pop 6 → 右邊界就是這個 2
pop 5 → 右邊界也是這個 2
停，因為 stack 剩下的 2 不比我高
一次走到一根矮柱子，就結算掉所有比它高的。
暴力解要每根柱子各自掃一遍才能得到的資訊，stack 一次遍歷就全部收集完。

思路鏈：

1. 暴力解的瓶頸是每根柱子都重複往左往右掃
2. 「找第一根比我矮的」是 monotonic stack 經典題型
3. 用兩個 pass 分別算 rightmost[] 和 leftmost[]
4. 面積 = heights[i] \* (rightmost[i] - leftmost[i] - 1)

觸發關鍵字：
看到「往某個方向找第一個比我大/小的元素」就想 monotonic stack。

== 3.One Pass Stack 解法（更精簡，但較難解釋）===

核心想法跟兩個 pass 版本一樣，但把左邊界藏在 start 裡，省掉第二個 pass。

逐步解釋：

1. 碰到比 stack 頂端矮的柱子時，pop 出來結算：
   - 被 pop 的柱子高度是 height，它的位置是 index
   - 右邊界就是當前的 i（因為 i 比它矮，擋住了）
   - 面積 = (i - index) \* height

2. start = index 的 trick：
   - 被 pop 掉的柱子比當前的 h 高
   - 所以當前這根矮柱子一定能往左延伸到被 pop 那根的位置
   - push 的時候記 (start, h) 而不是 (i, h)
   - 這樣左邊界就藏在 start 裡，不用另外算

3. 最後清空 stack：
   - 留在 stack 裡的柱子代表右邊沒有比它矮的
   - 所以右邊界就是陣列最尾端 len(heights)
   - 面積 = (len(heights) - i) \* h

=== 為什麼 stack 解法是 O(n) ===

每根柱子最多被 push 一次、pop 一次，總共 2n 次操作。
雖然裡面有 while loop，但整體還是 O(n)。

=== 面試建議 ===

用兩個 pass 的版本，左右邊界分開算，好解釋。
先講暴力解展示思路，再推導到 stack 優化，面試官看的是這個過程。

=== 反思 ===
一開始用錯用成雙指針，下次可以想
雙指針需要一個明確的起點，但這題你不知道最大長方形在哪
雙指針適合的場景是你有辦法縮小搜索範圍，例如：

-排序過的陣列找 two sum → 太大就縮右，太小就縮左
-Container With Most Water (11) → 矮的那邊往內縮，因為矮的留著不會更好

這些題的共同點是：每一步你都有明確的理由決定移哪邊，而且不會錯過答案。
這題不行，因為你站在任何位置，都沒有足夠資訊判斷該往左還是往右。你之前試過的 [0, 9] 就是最好的反例——往左看到 0 你覺得不該往左，但答案就在右邊那根，你的策略根本碰不到它。

問自己：「我每一步移動指針的理由是什麼？我能證明這樣不會錯過答案嗎？」
