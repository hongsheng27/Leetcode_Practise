
LeetCode 348 - Design Tic-Tac-Toe
====================================

題目：設計一個 n x n 的井字棋，每次 move 要判斷是否有人獲勝。

核心觀察：
  獲勝條件只有四種：某一行滿、某一列滿、主對角線滿、反對角線滿。
  不需要存整個棋盤，只需要追蹤這四種線的狀態。

解題思路：
  用分數代替實體陣列：player 1 下 +1，player 2 下 -1。
  維護四個計數器：
    h[row] → 每一行的分數
    v[col] → 每一列的分數
    d      → 主對角線的分數（row == col）
    rd     → 反對角線的分數（row + col == n - 1）
  某個計數器的絕對值 == n 時，代表同一個玩家佔滿了那條線，獲勝。

  這樣每次 move 只更新相關的計數器，O(1)。

對角線判斷條件：
  主對角線（\）：row == col（差相等）
  反對角線（/）：row + col == n - 1（和相等）

踩過的坑：
  1. 為了讓 move 達到 O(1)，用 +1/-1 分數取代存整個棋盤。
     直覺想法是存實體陣列再掃一遍判斷，但那樣每次 move 是 O(n)。
  2. 判斷獲勝時要加 abs，因為 player 2 的分數是負的，滿了會是 -n 不是 n。
  3. anti-diagonal 的條件是 row + col == n - 1

複雜度：O(1) per move，O(n) 空間。
