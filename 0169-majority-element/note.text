LeetCode 169 - Majority Element
題目：給一個陣列 nums，找出出現次數超過 n/2 的元素（保證一定存在）。

踩過的坑：
1.Counter 暴力計數可以 AC，但沒有利用到「超過 n/2」這個條件。
O(n) 時間 + O(n) 空間，面試官會追問能不能不用額外空間。
候選人不能用空字串或特殊值來表示「還沒選」。
2.如果 majority element 本身是 0，not me 會誤判，導致候選人一直被覆蓋。
應該用 cnt == 0 來決定換不換候選人，而不是看 me 的值。

解題思路：Boyer-Moore Voting Algorithm（Greedy）
核心直覺：majority element 出現超過一半，比其他所有元素加起來還多。
讓它跟非 majority 一對一抵消，抵消完一定是它剩下。
第一個要解決的問題：一開始不知道 majority 是誰。
不需要知道。先隨便選當前元素當候選人，開始計數。
遇到相同 +1，不同 -1。cnt 歸零就換候選人。

這是 Greedy 的選擇：每次歸零都直接換，不回頭。
第二個要解決的問題：cnt 歸零換人，為什麼不會錯過真正的 majority？
歸零代表前面那段候選人跟非候選人打平了。
majority 在整個陣列超過 n/2 次，扣掉一段打平的，
它在剩餘部分仍然是 majority，所以最後一定會勝出。
第三個要解決的問題：怎麼只用兩個變數完成？
me 記候選人，cnt 記票數。
cnt == 0 → 換候選人為當前元素。
n == me → cnt += 1，否則 cnt -= 1。
跑完整個陣列，me 就是答案。

複雜度：O(n) 時間，O(1) 空間。
