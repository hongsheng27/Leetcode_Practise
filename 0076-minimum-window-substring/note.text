
LeetCode 76 - Minimum Window Substring
========================================

題目：給字串 s 和 t，找 s 中包含 t 所有字元的最短 substring。
t 可能有重複字元，例如 t = "AAB" 代表需要 2 個 A 和 1 個 B。

解題思路：Variable Sliding Window

  第一個要解決的問題：怎麼知道窗口已經包含 t 的所有字元？
    直覺做法：兩個 map，一個記 t 的需求，一個記窗口內的字元，每次比對兩邊。
    優化：只用一個 map，從 t 的需求開始倒數。
    窗口碰到需要的字元就 -1，減到 0 代表這個字元夠了。
    不在 t 裡的字元直接跳過，不記錄。
    這樣省掉一個 map，也省掉比對兩個 map 的開銷。

  第二個要解決的問題：避免每次都掃整個 map 比對。
    用 have 變數記「有幾種字元已經滿足」。
    某字元的 count 從 1 降到 0 的瞬間 → have += 1（這個字元剛好夠了）。
    某字元的 count 從 0 升到 1 的瞬間 → have -= 1（這個字元又不夠了）。
    have == len(map) 時，代表所有字元都滿足，開始縮左邊界找更短的。

  第三個要解決的問題：記錄答案要高效。
    不要每次都切字串（O(n)），改成記 (start, end) 的 index。
    初始值用 (0, float("inf"))，任何長度都比 inf 小，第一次一定會更新。
    最後才切一次字串。如果 inf 沒被更新過代表沒找到，回傳 ""。

窗口操作流程：
  擴右邊界 → 碰到需要的字元就倒數，減到 0 就 have += 1
  have 全滿 → 縮左邊界，記錄更短的答案，碰到需要的字元就加回去，升到 1 就 have -= 1

複雜度：O(n) 時間，O(1) 空間（最多 52 個字元）。
