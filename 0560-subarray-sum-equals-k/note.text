LeetCode 560 - Subarray Sum Equals K

題目：給一個整數陣列和一個整數 k，找出和為 k 的連續子陣列有幾個。

解題思路：Prefix Sum + HashMap
第一個要解決的問題：暴力解太慢。
枚舉所有 (i, j) 配對，檢查每個子陣列的和是不是 k → O(n²)。
nums 長度最大 20,000，n² = 4 億次操作，TLE。
第二個要解決的問題：怎麼不用枚舉所有配對？
用 prefix sum，子陣列和 = prefix[j+1] - prefix[i]。
我們在找 prefix[j+1] - prefix[i] == k，移項得：
prefix[i] == prefix[j+1] - k
也就是說：走到某個位置時，想知道「之前有沒有出現過值等於 curSum - k 的 prefix sum」。
「之前出現過幾次某個值」→ 用 HashMap 記錄每個 prefix sum 出現的次數。
第三個要解決的問題：為什麼用 map 不用 set？
set 只能告訴你「有沒有出現過」。
但同一個 prefix sum 值可能出現很多次，每一次都對應一個不同的子陣列。
所以需要知道「出現過幾次」→ HashMap。
第四個要解決的問題：順序和初始值。
順序：先查再存。保證只跟「之前的」prefix sum 配對，不會跟自己配對。
初始值：map = {0: 1}，代表 prefix sum = 0 出現過 1 次。
不加這個，當子陣列從 index 0 開始剛好和為 k 時會漏掉。
這就是 303 學的補 0 技巧，在這裡用 map 的形式體現。

複雜度：O(n) 時間，O(n) 空間。
踩過的坑：
一開始先建完整個 prefix 陣列再跑雙迴圈 → 還是 O(n²)，沒用到 map。
prefix sum 的價值不在於建陣列，在於搭配 map 做 O(1) 查詢。
想用 set 而不是 map → set 只知道有沒有，不知道幾次，會少算。
先存再查 → 會把自己跟自己配對，或 index 越界。一定要先查再存。
忘記 {0: 1} → 子陣列從頭開始且和為 k 的情況會漏掉。

思路鏈：
暴力解枚舉所有子陣列 → O(n²) TLE
子陣列和 = prefix[j+1] - prefix[i] → 移項變成查找問題
「之前有沒有出現過某個值」→ HashMap
要知道「幾次」不只是「有沒有」→ map 不是 set
先查再存 + {0: 1} 處理邊界

觸發關鍵字：
看到「子陣列的和等於某個值」「連續子陣列」→ prefix sum + hashmap。
這個組合是面試超高頻 pattern，523、525 也是同一個套路。
