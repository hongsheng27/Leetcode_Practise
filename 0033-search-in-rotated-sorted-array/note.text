筆記：33. Search in Rotated Sorted Array
題目:有序陣列在某個未知點被旋轉（如 [4,5,6,7,0,1,2]），給定 target 找 index，找不到回傳 -1，要求 O(log n)。
解題邏輯
核心觀察：旋轉陣列從中點切開後，左右兩半必有一半是完全有序的。
步驟：
計算 m = (l + r) // 2
若 nums[m] == target → 直接回傳
判斷哪半有序：nums[l] <= nums[m] 成立 → 左半有序，否則右半有序
在有序的那半用「兩端夾擊」判斷 target 是否在其中
在 → 搜該半；不在 → 搜另一半


踩過的坑
1.以為要先比較 target 和 nums[m] 的大小 → 在旋轉陣列中這不足以判斷方向，因為「比 m 小」可能在左半也可能在右半。
正確順序：先找有序那半，再用範圍判斷。
判斷有序半時忘記用 <=：nums[l] <= nums[m] 的等號很重要，當 l == m 時（搜尋空間剩兩個元素），左半「只有一個元素」也算有序。
2.範圍判斷的開閉：因為迴圈開頭已經處理 nums[m] == target，所以 nums[l] <= target < nums[m] 用嚴格小於 nums[m]，避免重複比較。
3.邊界用 m 而非 m ± 1：會無窮迴圈。閉區間範式中永遠用 m ± 1。

複雜度
時間：O(log n)，每次排除一半
空間：O(1)
