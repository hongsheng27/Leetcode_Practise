71. Simplify Path
題目概述： 給 Unix 風格的絕對路徑如 /home//foo/../bar，簡化成標準形式：去除多餘 /、解析 .（當前目錄）和 ..（上一層），回傳簡化後的路徑。
解題邏輯： 用 split("/") 切成段，用 stack 處理：

空字串或 . → 跳過
.. → 如果 stack 非空就 pop
其他 → append

最後 "/" + "/".join(stack) 組回路徑。
踩過的坑：

一開始用字元層級處理，想在迴圈裡偵測 /、.、..、連續斜線、開頭結尾 edge case，規則爆炸。正確處理單位是段（name），不是字元。先 split 成段再處理，字元層級的 edge case 全部自動被 split 吃掉。
split("/") 會自動處理連續斜線（中間產生空字串）、開頭/結尾斜線。只需處理 ""、.、..、正常名稱四種 case。

一般化觀念：

先問處理單位： 字元？段？區間？單位不匹配時會在迴圈裡手刻邊界，邏輯爆炸。Python 有 split / strip / re.findall 能切成正確單位，先切再處理永遠比邊讀邊判斷乾淨。
列「要丟掉的」比列「要保留的」簡單： 用 != 濾掉異常 case，其他全收，比用 == 列舉所有正常情況好寫。

複雜度： O(n) time / O(n) space
