0199. Binary Tree Right Side View
題目概述
想像你站在二元樹的右側，由上到下看過去，回傳每層你能看到的那個 node 的值。每
層被擋住的 node 看不到，只有最右邊的能看見。注意「最右」是指那一層位置最右邊的 node，
不一定是右子節點——如果某層只有左子節點，那個左子就是該層的「最右」。
例：根為 1，左子 2、右子 3；2 沒有左子但有右子 5；3 沒有左子但有右子 4。從右側看依序看到 1、3、4。

解題邏輯
BFS 一層一層走，每層取最右邊。核心 insight：進入 while 時 q 剛好是「當前層由左到右排列」→ q[-1] 就是最右，可以提前讀不用等 for loop 跑完。

踩過的坑
1.append 順序決定左右視角：先 left 後 right → q[-1] 是最右。寫反就變 left side view。
2.q[-1] 必須在 for loop 前讀：進 for loop 後 q 會混入下一層的 node，q[-1] 不再是當前層最右。

複雜度
Time O(n)，每個 node 進出 queue 各一次。
Space O(w)，w = 樹的最大寬度。最壞情況（balanced tree 的最底層）w ≈ n/2，
所以 worst case 是 O(n)；skewed tree 則 w = 1。
