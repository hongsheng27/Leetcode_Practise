767. Reorganize String
     ======================

題目概述：
給一個字串 s，重新排列字元，讓相同字元不會相鄰。
如果無法做到，回傳空字串。

解題邏輯：
使用 max heap。

heap 存：
[-剩餘次數, 字元]

每次取出剩餘次數最多的字元，加入答案。

為了避免連續取到相同字元：
目前使用的字元先暫存在 preStatus。
等下一輪已經選完其他字元後，再把上一輪的 preStatus 放回 heap。

因為這題只要求相同字元不能相鄰，
所以只需要暫存上一輪字元，不需要 cooldown queue。

最後如果答案長度小於原字串長度，
代表剩餘字元無法合法放入，回傳空字串。

踩過的坑：
1. 一開始 heap 只存 count。
   但最後需要組合新字串，所以 heap 必須同時存 char：

   [-count, char]

2. 一開始使用 queue 處理 cooldown。
   但這題只需要隔開一個位置，所以只要暫存上一輪字元即可。

複雜度：
設：
n = 字串長度
k = 不同字元數量

時間複雜度：O(n log k)
每個字元最多進出 heap 一次。
heap 大小最多為 k。

空間複雜度：O(k)
heap 和 count 最多存 k 種字元。
