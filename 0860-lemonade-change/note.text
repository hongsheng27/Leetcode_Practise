860. Lemonade Change
題目概述： 檸檬攤賣 $5 檸檬水，顧客付 $5、$10 或 $20。按順序服務每個人，問能不能給所有人正確找零。初始沒零錢。

解題邏輯： Greedy + 追蹤零錢。

Greedy 策略（局部最優）：
收 $20 時，優先用 1×$10 + 1×$5 找零，不夠才用 3×$5。
為什麼？因為 $5 更萬用（$10 和 $20 都能用），$10 只能用來找 $20。保留 $5 的靈活性，別浪費在能用 $10 解決的地方。
證明正確性（反例檢查）：
如果「收 $20 優先用 3×$5」→ 可能把 $5 用光，後來 $10 無法找零（$10 需要 1×$5）。

Greedy 解題框架（這題的應用）：

題目要什麼： 能不能給所有人找零
局部最優： 每次收錢時的找零策略 = 優先用大面額，保留小面額靈活性
為什麼對： $5 更萬用，找到反例（優先用 $5 會失敗的 case）證明策略正確
追蹤狀態： $5 和 $10 的數量（$20 不用追蹤，收了也找不出去）


踩過的坑：
1.Greedy 沒有統一程式碼模板，每題的「局部最優」定義不同。模板是思維框架：進入每題前問「貪心策略是什麼、為什麼對、需要追蹤什麼」，不是背固定的 code pattern。
2. 可以有 5,5,5 或者 5, 10, 可以下去backtracking, 但是greedy 可以直接選擇 5, 10 就是最佳答案所以不用跑bscktrack
複雜度： O(n) time / O(1) space
