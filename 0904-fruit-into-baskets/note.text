LeetCode 904 - Fruit Into Baskets
===================================

題目：給一排果樹，你有兩個籃子，每個籃子只能裝一種水果。
找最長的連續子陣列，使得裡面最多只有 2 種水果。

本質就是：最長的子陣列，最多包含 2 種不同的值。

解題思路：Variable Sliding Window（找最長）

  用 map 記窗口內每種水果的數量。
  右邊界擴展時加入 map。
  當 len(map) > 2（超過 2 種）→ 縮左邊界，數量減到 0 就 del 掉。
  每一步更新 res = max(res, r - l + 1)。

踩過的坑：

  1. 直接套了 76 題的模板，但方向搞反了。
     76 是「條件滿足時縮窗口，找最短」。
     這題是「條件違反時縮窗口，找最長」。
     下次先分清楚找最短還是找最長，再決定模板。

  2. 用了 have 變數追蹤種類數，但這題不需要。
     76 用 have 是因為 count 裡的值代表「還差幾個」，不能刪。
     這題的 count 單純記數量，歸零就 del，用 len(count) 就能追蹤種類數。

  3. 用 sum(count.values()) 算窗口大小，但窗口大小就是 r - l + 1，不用加總。
  4. 不知道計算 res 要放在哪

Sliding Window 模板選擇：
  找最短 → 條件滿足時縮窗口（76 的模式）
  找最長 → 條件違反時縮窗口（這題的模式）

複雜度：
O(n) 時間
O(1) 空間（最多 3 個 key）。
