98. Validate Binary Search Tree

題目: 給一棵 binary tree，判斷它是不是合法的 BST。

重點是：BST 不是只看「左孩子 < 自己 < 右孩子」，而是要看整個子樹範圍。
例如右子樹裡的所有節點，都必須大於 root；左子樹裡的所有節點，都必須小於 root。

經典反例：

      5
     / \
    1   6
       / \
      4   7

4 雖然小於 6，但它在 5 的右子樹，所以它應該要大於 5。因此這不是合法 BST。

解題邏輯
用 DFS 搭配「上下界」。
每個節點都帶著一個合法範圍：
lower < node.val < upper
一開始 root 沒有限制，所以範圍是負無限到正無限。
往左子樹走時，因為左子樹所有值都要小於目前節點，所以「上界」變成目前節點值。
往右子樹走時，因為右子樹所有值都要大於目前節點，所以「下界」變成目前節點值。

踩過的坑
1.這題一開始容易只想到遞迴左右子樹：
2.左邊要小於自己，右邊要大於自己
但真正重要的是：目前 node 自己也要先檢查是否符合祖先傳下來的限制。

我當時的思考順序是：

先想到上下界怎麼傳
再想到左右遞迴
再想到 return value
最後才想到要檢查自己

這題之後可以記成：
只要 dfs 有帶 extra info，例如 lower / upper，
就要問：目前 node 是否要先跟這些 info 做檢查？


複雜度
時間複雜度：O(n)
每個節點最多檢查一次。
空間複雜度：O(h)
h 是樹高，來自遞迴 call stack。平衡樹是 O(log n)，最壞情況退化成 linked list 是 O(n)。
