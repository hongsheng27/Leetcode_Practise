22 Generate Parentheses

題目解釋：
給定 n，生成所有合法的括號組合。例如 n=3 → ["((()))","(()())","(())()","()(())","()()()"]。
核心概念是 backtracking，在每一步決定放 "(" 或 ")"，透過條件剪枝確保只產生合法組合。

解題思路：

1. 長度固定為 2n，每個位置只有兩種選擇：放 "(" 或 ")"
2. 不能暴力窮舉所有組合再過濾，要在建構過程中就確保合法 → backtracking + 剪枝
3. 合法的關鍵條件只有兩個：
   - openN < n → 還能放 "("（左括號不能超過 n 個）
   - closedN < openN → 還能放 ")"（右括號不能超過已有的左括號數）
4. 終止條件：closedN == openN == n，代表 2n 個字元都放完且合法
5. 用 stack 組字串，每次遞迴回來要 pop（回溯）

踩過的坑：

1. 一開始想不到 backtrack 需要哪些參數。其實就追蹤 openN 和 closedN 兩個計數器就夠了，因為所有決策都是基於「目前放了幾個左括號、幾個右括號」來判斷。
2. 沒有馬上想到兩個關鍵的合法條件（openN < n / closedN < openN）。記法：左括號看上限 n，右括號看左括號數量，右永遠不能超過左。

複雜度：
時間 | O(4^n / √n)，第 n 個 Catalan number (之後再研究)
空間 | O(n)，遞迴深度 + stack 長度
